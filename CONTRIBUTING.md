# Contributing to DICOM Vision

Thank you for your interest in contributing to DICOM Vision! We welcome contributions from the community to help improve this project.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please create an issue on GitHub with:
- A clear description of the bug.
- Steps to reproduce the issue.
- Expected behavior vs actual behavior.
- Any relevant logs or screenshots.

### 2. Suggesting Enhancements
Have an idea to make DICOM Vision better? We'd love to hear it! Please open an issue and:
- Explain why the enhancement is needed.
- Describe how it should work.

### 3. Submitting Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes and ensure the code follows established patterns.
4. Run tests (if applicable) and verify your changes.
5. Commit your changes with a descriptive message:
   ```bash
   git commit -m "Add feature: your feature description"
   ```
6. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. Open a Pull Request on the main repository.

## Development Guidelines

- **Code Style**: 
  - For Flutter: Follow the [official Dart style guide](https://dart.dev/guides/language/evolutionary-style).
  - For Python: Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).
- **Commit Messages**: Keep them clear and concise.
- **Documentation**: Update the README if your changes introduce new features or change existing workflows.

## Questions?
If you have any questions, feel free to open an issue or contact the maintainers.

Happy coding!
